1 # OpenCL CPU/GPU Support {#opencl_support}
2 
3 [OpenCL](https://www.khronos.org/opencl/) is an open-source framework for writing programs that utilize a platform with heterogeneous hardware. Stan uses OpenCL to design the GPU routines for the Cholesky Decomposition and it's derivative. Other routines will be available in the future. These routines are suitable for programs which require solving large `NxM` matrices (`N>600`) such as algorithms that utilize large covariance matrices.
4 
5 # Requirements
6 
7 Users must have suitable hardware (e.g. Nvidia or AMD gpu), valid OpenCL driver, SDK and a suitable C/C++ compiler installed on their computer.
8 
9 # Installation
10 
11 ## Linux
12 The following guide is for Ubuntu, but it should be similar for any other Linux distribution. You should have the GNU compiler suite or clang compiler installed beforehand.
13 
14 Install the Nvidia CUDA toolkit and clinfo tool if you have an Nvidia GPU
15 ```bash
16 apt update
17 apt install nvidia-cuda-toolkit clinfo
18 ```
19 
20 Those with AMD devices can install the OpenCL driver available through
21 
22 ```bash
23 apt install -y libclc-amdgcn mesa-opencl-icd clinfo
24 ```
25 
26 If your device is not supported by the current drivers available you can try Paulo Miguel [PPA](https://laanwj.github.io/2016/05/06/opencl-ubuntu1604.html)
27 
28 ```bash
29 add-apt-repository ppa:paulo-miguel-dias/mesa
30 apt-get update
31 apt-get install libclc-amdgcn mesa-opencl-icd
32 ```
33 
34 ## MacOS
35 Mac's should already have the OpenCL driver installed if you have the appropriate hardware.
36 
37 Note that if you are building on a mac laptop you may not have a GPU device. You can still use the OpenCL routines for parallelization on your CPU.

39 ## Windows
40 Install the latest [Rtools](https://cran.r-project.org/bin/windows/Rtools/) suite if you don't already have it. During the installation make sure that the 64 bit toolchain is installed. You also need to verify that you have the System Enviroment variable `Path` updated to include the path to the g++ compiler (`<Rtools installation path>\mingw_64\bin`).
41 
42 If you have a Nvidia card, install the latest [Nvidia CUDA toolkit](https://developer.nvidia.com/cuda-toolkit).
43 AMD users should use [AMD APP SDK](http://amd-dev.wpengine.netdna-cdn.com/app-sdk/installers/APPSDKInstaller/3.0.130.135-GA/full/AMD-APP-SDKInstaller-v3.0.130.135-GA-windows-F-x64.exe).
44 
45 Users can check that their installation is valid by downloading and running clinfo.
46 - [download clinfo.exe](https://ci.appveyor.com/api/projects/oblomov/clinfo/artifacts/clinfo.exe?job=platform%3a+x64)

48 ## Setting up the Math Library to run on a GPU
49 
50 To turn on GPU computation:
51 
52 1. Check and record what device and platform you would like to use with [clinfo](https://github.com/Oblomov/clinfo); you will the platform and device index such as the printout below
53 
54 ```bash
55 clinfo -l
56 # Platform #0: Clover
57 # Platform #1: Portable Computing Language
58 # `-- Device #0: pthread-AMD Ryzen Threadripper 2950X 16-Core Processor
59 # Platform #2: NVIDIA CUDA
60 # +-- Device #0: TITAN Xp
61 # `-- Device #1: GeForce GTX 1080 Ti
62 ```
63 
64 2. In the top level of the math library, open a text file called make/local. if it does not exist, create one.
65 3. Add these lines to the make/local file:
66 
67 ```bash
68 STAN_OPENCL=true
69 OPENCL_DEVICE_ID=${CHOSEN_INDEX}
70 OPENCL_PLATFORM_ID=${CHOSEN_INDEX}
71 ```
72 
73 where the user will replace ${CHOSEN_INDEX} with the index of the device and platform they would like to use. In most cases these two will be 0. If you are using Windows append the following lines at the end of the make/local file in order to link with the appropriate OpenCL library:
74 
75 * Nvidia
76 ```make
77 CC = g++
78 LDFLAGS_OPENCL= -L"$(CUDA_PATH)\lib\x64" -lOpenCL
79 ```
80 
81 * AMD
82 ```make
83 CC = g++
84 LDFLAGS_OPENCL= -L"$(AMDAPPSDKROOT)lib\x86_64" -lOpenCL
85 ```
86 
87 ## Running Tests with OpenCL
88 
89 Once you have done the above step, `runTests.py` should execute with the GPU enabled. All tests will match the phrase `*_opencl_*` and tests can be filtered such as
90 
91 ```bash
92 ./runTests.py test/unit -f opencl
93 ```

95 ## Using the OpenCL backend
96 
97 We currently have support for the following methods
98 
99 - bernoulli_logit_glm
100 - cholesky_decompose
101 - categorical_logit_glm
102 - gp_exp_quad_cov
103 - mdivide_right_tri
104 - mdivide_left_tri
105 - multiplication
106 - neg_binomial_2_log_glm
107 - normal_id_glm
108 - ordered_logistic_glm
109 - poisson_log_glm
110 
111 In cmdstan, an example model is provided in `examples/GP/`, which uses OpenCL Cholesky decomposition. You can check if your OpenCL configuration works by trying to build it.
112 * Linux and MacOS: `make examples/GP/gp3`
113 * Windows: `make examples/GP/gp3.exe`
